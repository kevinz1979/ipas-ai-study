費曼學習法：每一題都要能教人

1. 白話定義

不用術語解釋概念。例如：「過擬合就是學生只背考古題，換題目就不會；模型太記住訓練資料，遇到新資料表現變差。」

2. 找卡點

解釋時停頓、只能背名詞、分不清相近概念，就是缺口。把缺口寫成一個問題，而不是模糊地重看整章。

3. 回官方資料補洞

查學習指引、樣題與公告試題；補上定義、流程、適用情境與反例。

4. 縮成教學卡

每張卡只放：「一句話定義、生活例子、何時用、容易混淆的概念」。隔天閉卷講 60 秒。

示範：如何解釋 RAG？

白話：RAG 像開卷考。模型先從可信文件找資料，再依找到的內容回答，減少憑記憶亂答。
何時用：公司內規、產品手冊、常更新且需引用來源的知識問答。
風險：找錯文件、文件過期、切段不佳，仍可能造成錯誤答案。
考場辨識：題目出現「私有文件、檢索、引用、降低幻覺」，優先聯想 RAG。

示範：如何解釋 Precision 與 Recall？

Precision：被模型判為「是」的案件中，真正是的比例；重點是少誤報。
Recall：所有真正「是」的案件中，被抓到的比例；重點是少漏掉。
例子：癌症篩檢通常重視 Recall，垃圾信過濾常會重視 Precision 與使用體驗的平衡。

刷題與錯題規則

做題三輪

第一輪：不計時，逐題寫理由。
第二輪：限時，模擬考場節奏。
第三輪：只做錯題與相似概念，直到能口述排除其他選項。

高頻辨識框架

看到情境題，依序問：目標是什麼？資料足夠嗎？選什麼技術？用何指標？有何風險與治理？中級題尤其常考這條決策鏈。

用免費平台建立本站

最簡單：GitHub Pages

  1. 在 iPad 安裝或以 Safari 開啟 GitHub，免費註冊帳號。
  2. 按「New repository」，名稱填 ipas-study，選 Public，建立儲存庫。
  3. 新增檔案，檔名必須是 index.html，把本頁原始碼完整貼上後按 Commit changes。
  4. 進入儲存庫的 Settings → Pages → Build and deployment，Source 選「Deploy from a branch」，Branch 選 main 與 /(root)，按 Save。
  5. 等待數分鐘後，GitHub 會顯示網站網址；用 Safari 開啟，再用「分享 → 加入主畫面」。

更新方式：GitHub 開啟 index.html → Edit → 修改文字 → Commit。幾分鐘後網站自動更新。

更適合純 iPad 操作：Netlify Drop

  1. 把這份內容存成 index.html（可使用 iPad「檔案」App 或任一文字編輯器）。
  2. Safari 前往 Netlify Drop，登入免費帳戶後上傳該 HTML 檔。
  3. 平台會提供公開網址；之後要更新時重新上傳新版本。

建議：要長期維護與保留版本，選 GitHub Pages；只想最快把單頁公開，選 Netlify Drop。不要把身分證、准考證號、私人筆記或付費教材上傳公開網站。
