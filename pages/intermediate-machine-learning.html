讀書定位： 中級不是只背名詞。你要能從題目中的商業問題、資料條件與風險， 推論適合的資料處理、模型、評估指標與改善做法。 遇到程式題時，目標是讀懂程式「在做什麼」，不只是背語法。

考科範圍地圖

1. 問題與資料

釐清分類、迴歸、分群或降維問題；辨識標籤、特徵、訓練資料與資料品質問題。

2. 資料前處理

缺失值、重複值、離群值、編碼、標準化、正規化、資料切分與資料洩漏。

3. 模型與演算法

監督式學習、非監督式學習、分類、迴歸、決策樹、集成方法、分群與降維。

4. 評估與調校

混淆矩陣、Accuracy、Precision、Recall、F1-score、ROC-AUC、MAE、MSE、RMSE。

5. 泛化與風險

欠擬合、過擬合、交叉驗證、正則化、類別不平衡、偏誤、公平性與可解釋性。

6. 部署與維運

模型版本、資料漂移、概念漂移、監控、再訓練、隱私與模型治理。

機器學習完整流程

步驟 1｜定義問題
先問：「我要預測數值、判斷類別，還是找出群體？」例如預測房價是迴歸；判斷是否違約是分類；客戶分眾是分群。
步驟 2｜盤點資料
確認資料來源、資料量、標籤品質、欄位意義、個資與授權限制。資料不可靠時，模型再複雜也無法補救。
步驟 3｜清理與轉換
處理缺失值、重複值與異常值；將文字類別轉數值；依模型需要進行縮放或標準化。
步驟 4｜切分資料
訓練集用來學習；驗證集用來比較模型和調整超參數；測試集留到最後，評估模型面對未見資料的表現。
步驟 5｜建立基準與模型
先用簡單基準模型，再比較較複雜模型。選擇模型時考慮準確度、可解釋性、運算成本、延遲與公平性。
步驟 6｜評估與改善
依任務與錯誤成本選指標；檢查過擬合、資料洩漏、類別不平衡與不同群體的表現。
步驟 7｜部署、監控、再訓練
上線後持續看資料漂移與模型表現；設定告警、版本與回退機制，必要時以新資料再訓練。

評估指標：先看錯誤成本

Accuracy

白話：全部題目中，答對的比例。

注意：類別嚴重不平衡時可能誤導。例如 99% 都是正常交易，全部猜正常也有 99% Accuracy。

Precision

白話：模型說「是」的案件中，真正是的比例。

重點：想減少誤報時特別重要。

Recall

白話：所有真正「是」的案件中，被模型抓到的比例。

重點：想減少漏報時特別重要，例如疾病初篩。

F1-score

白話：同時兼顧 Precision 與 Recall 的平衡指標。

重點：資料不平衡且兩種錯誤都在意時常使用。

MAE、MSE、RMSE

白話：用於迴歸任務，衡量預測數字與真實數字相差多遠。

辨識：MSE 與 RMSE 對大誤差更敏感；MAE 比較直觀。

混淆矩陣

白話：把預測和真實結果交叉對照，得到 TP、TN、FP、FN。

考場：先辨識「誤報 FP」與「漏報 FN」，再判斷該優先改善哪個。

費曼教學卡：閉卷講 60 秒

監督式學習 vs 非監督式學習

監督式：像有答案的練習題，資料已經標好類別或數值，模型從中學規則。

非監督式：像拿到一堆沒有標記的照片，模型自己找出相似群體或結構。

考場關鍵：有標籤、預測答案，偏向監督式；無標籤、客群分組，偏向非監督式。

過擬合 vs 欠擬合

過擬合：像只背熟練習題，訓練資料很準，但遇到新題目就失常；通常模型太複雜或資料太少。

欠擬合：像只學到很粗淺的規則，連練習題也答不好；通常模型太簡單或特徵不足。

改善：過擬合可考慮更多資料、正則化、交叉驗證、降低複雜度；欠擬合可改善特徵或提高模型表達能力。

資料洩漏是什麼？

白話：模型在考試前偷看到答案線索，導致離線成績很好，上線後卻失效。

例子：用「是否已完成理賠」來預測「是否詐欺」，但理賠狀態是在事件發生後才知道。

考場關鍵：題目出現未來資訊、目標欄位衍生資訊，優先懷疑資料洩漏。

交叉驗證為什麼有用？

白話：不是只用一次切分決定模型，而是輪流把不同部分當驗證資料，多次測試模型穩定度。

用途：資料不多時，更有效利用資料來比較模型與調整超參數。

程式判讀與刷題規則

看到程式，先找動詞

  • read_csv：讀取資料。
  • head、info、describe：檢視資料。
  • isnull：檢查缺失值。
  • dropna、fillna：處理缺失值。
  • train_test_split：切分訓練與測試資料。
  • fit：用訓練資料學習。
  • predict：產生預測結果。
  • score、accuracy_score：評估模型。

每題四問

  1. 這題的目標是分類、迴歸、分群還是降維？
  2. 資料是否有標籤？是否有缺失、不平衡或洩漏？
  3. 哪個模型或前處理最符合問題限制？
  4. 錯誤成本是誤報較高，還是漏報較高？應看哪個指標？
